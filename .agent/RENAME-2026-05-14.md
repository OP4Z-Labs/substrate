## Decisions taken

1. **Adapter naming: `@op4z/substrate-adapter-X`.** The brief said
   "verify exact names and rename to `@op4z/substrate-adapter-{...}`"
   ŌĆö confirmed and applied across all four adapters. The sibling-name
   shape (matching `@op4z/substrate`) makes the npm-search discovery
   story coherent.

2. **Combined commit, not six.** As noted above ŌĆö the phases couple
   tightly (Phase 1 source code depends on Phase 2 template file
   renames), and splitting them produced intermediate states with
   failing gates. The brief's "all gates green after each phase
   commit" is the binding constraint; one commit honors it.

3. **`SUBSTRATE_VERSION` not `OP4Z_SUBSTRATE_VERSION`.** The identifier
   represents the version of `@op4z/substrate`. The `@op4z/` namespace
   doesn't need to be encoded in the constant name; the constant is
   module-local and the package name is the namespace. Mirrors how
   `commander` exports a `Command` class, not `COMMANDER_Command`.

4. **Telemetry `SCHEMA_VERSION` bumped to 2.** Mandated by the brief
   when forbidden fields change. The seed data in
   `telemetry-commands.test.ts` left at `v: 1` to exercise legacy-event
   parsing tolerance ŌĆö the show/export tools intentionally don't
   schema-validate, so this remains a useful test of "old events on
   disk still parse cleanly."

5. **`docs-site` package.json name unscoped (`substrate-docs-site`,
   not `@op4z/substrate-docs-site`).** The docs site is private
   (`"private": true`) and never publishes. An unscoped name keeps
   the workspace selectors clean (`npm run --workspace
   substrate-docs-site docs:build`). The scope would only matter
   on npm publish.

6. **Templates-history/1.0.0 snapshot reflects substrate names.**
   Because v1.0.0 IS the inaugural `@op4z/substrate` release ŌĆö
   there was never a `cadence@1.0.0`. The brief's locked policy:
   the snapshot at version N must reflect what version N actually
   shipped. v0.5 frozen as cadence-named (that historical accuracy
   is the merge-anchor's value); 1.0 is substrate-named.

7. **The repo directory `/home/beaug/dev/public/cadence/` stays
   named "cadence" during the agent run.** Renaming the working
   directory mid-agent would break the bash CWD across calls.
   Beau will manually `mv ~/dev/public/cadence ~/dev/public/substrate`
   after this run completes. The git remote (when added) will be
   `github.com/op4z/substrate`.

8. **Repository URLs pinned to `github.com/op4z/substrate`.** The
   repo doesn't exist yet ŌĆö these are placeholders for when Beau
   pushes. The `homepage`, `repository.url`, and `bugs.url` fields
   in `packages/substrate/package.json` all point at the canonical
   future location. Internal docs (`README.md`, `CHANGELOG.md`,
   case studies) use the same URL.

9. **HANDOFFs preserved as historical record.** `.agent/HANDOFF-*.md`
   were not retroactively rewritten. They document what shipped
   when; rewriting them would erase the project's own provenance
   trail. Only `HANDOFF-2026-05-14-v1.md` got a one-paragraph header
   note pointing at this rename ŌĆö readers landing on the v1.0 GA
   handoff need the breadcrumb that the package name evolved
   immediately after.
