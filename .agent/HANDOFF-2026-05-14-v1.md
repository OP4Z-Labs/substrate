## Completed in this run

### Phase A — RULES.yaml detector runtime

The single highest-leverage change in v1.0. Cadence now actually
**runs** the rules in `cadence/RULES.yaml`, where v0.x just listed
them.

- New subsystem at `src/audit/`:
  - `types.ts` — locked types for `AuditReport`, `RuleDefinition`,
    `Finding`, `Detector` (`ripgrep | script | composite`).
  - `rules.ts` — loader with strict-mode validation, duplicate-ID
    detection, legacy `shell` deprecation handling.
  - `runner.ts` — orchestrates rules. Phase 1 runs non-composites,
    phase 2 evaluates composites against the sub-results.
  - `report.ts` — atomic-write Markdown report + JSON sidecar +
    `_trend.jsonl` journal.
  - `diff-paths.ts` — `git diff --name-only HEAD` + untracked union
    for `--diff` mode.
  - `trend.ts` — reads `_trend.jsonl` for `--trend`.
  - `index.ts` — public re-exports.

- Three detector implementations under `src/audit/detectors/`:
  - **`ripgrep.ts`** — shells out to `rg` (argv-array, no
    shell-interpolation) when on PATH; falls back to a Node-only
    regex scan otherwise. Honors default excludes
    (`node_modules`, `.git`, etc.) and supports
    `caseSensitive` / `fixedString` / `multiline`.
  - **`script.ts`** — launches a Node `worker_threads.Worker` with
    an empty env, the user script loaded via `import()`. 30-second
    default timeout (cap 5 min) enforced via `worker.terminate()`.
    Worker harness at `script-worker.ts` exposes a `DetectorContext`
    with filesystem reads constrained to `repoRoot` (EPERM otherwise).
  - **`composite.ts`** — combines other rules via `all` / `any` / `none`
    operators. Evaluated after sub-rules.

- CLI surface:
  - `cadence audit` runs every rule.
  - `--rule <id>` runs one.
  - `--diff` restricts ripgrep detectors to staged-diff files.
  - `--trend` reads the journal.
  - `--rules-path <path>` overrides RULES.yaml location.
  - `--strict` makes unknown YAML fields fatal.
  - `--no-report` skips writing report files (CI / scripting).
  - `--json` emits structured stdout.

- Report artefacts:
  - `cadence/audits/<scope>-YYYY-MM-DD.md` (human Markdown)
  - `cadence/audits/<scope>-latest.json` (machine sidecar)
  - `cadence/audits/_trend.jsonl` (append-only journal)

- **Contract doc:** `docs/audit-runtime.md` (the authoritative
  reference for rule authors).

**Tests:** +32 unit (`tests/audit-runtime.test.ts`,
`tests/audit-script-detector.test.ts`) + 7 integration

### Design decisions taken

1. **Detector contract is locked but extensible.** Three concrete
   types ship (ripgrep, script, composite). Adding a fourth is a
   v1.x additive change. Locked in `src/audit/types.ts` head
   comment.

2. **Script sandbox is light, not paranoid.** Filesystem reads
   constrained to `repoRoot`; env blank to avoid secret leakage;
   timeout enforced via `worker.terminate()`. Network access is
   NOT blocked — rule authors run in a trusted context (their own
   RULES.yaml). Documented in `docs/audit-runtime.md`.

3. **Atomic write is local-only.** Same-directory tmp file + rename;
   no `fsync` of the directory entry. Pragmatic trade-off
   documented in the helper's head comment.

4. **Telemetry stays local by default.** The
   `--telemetry-endpoint` opt-in is the only mechanism to send
   events anywhere. There is no cadence-hosted collector; the
   maintainer's stance is "you can ship events anywhere you want,
   we don't host."

5. **`shell` detector deprecated, not removed.** v0.x RULES.yaml
   files keep loading; the runtime marks `shell` rules as
   `skipped: true` with a deprecation note. Migration path is
   documented in `docs/migration-from-0.x.md`.

6. **Standards-doc shape is consistent across the catalog.** All
   21 docs follow Scope / Rules / Examples / Rationale. The
   uniformity makes the catalog easier to navigate; the cost is
   that a doc that doesn't fit (e.g. a quick-reference list) has
   to bend to the shape.

7. **v1.0 version bump landed in Phase C+D commit.** Bumping
   earlier would have produced misleading `cadence --version`
   output during the Phase A and B work. Bumping at the end
   matches the "all gates green for v1.0" milestone.

8. **Templates-history `1.0.0/` is a snapshot of current
   `templates/`.** Per the brief: "every shipped minor version
   must snapshot templates/ into templates-history/<NEW_VERSION>/
   BEFORE the version bump." Done.

9. **`cadence uninstall` is conservative by default.** Preserves
   user-modified files via manifest hash check; `--force` is the
   knob for "I really mean it." The plan is shown by default and
   only acted on with `--yes`. Documented in `--help` output.

10. **Programmatic API stability is the v1.0 promise.** Within
    v1.x, only additive changes. Breaking changes wait for v2.0.
    Documented in `docs/programmatic-api.md`.

### Working directory and state

- **Working directory:** `~/dev/public/cadence`. v1.0

- `packages/cadence/templates-history/1.0.0/` is the new snapshot
  shipping with v1.0. Future minor versions must add their own.

### Patterns established for v1.x development

- **Detector contract is locked.** Adding a new detector type:
  (a) add to the type union in `types.ts`, (b) add a runner under
  `detectors/`, (c) wire into `runner.ts`'s `runSingleRule()`,
  (d) document in `docs/audit-runtime.md`. Use this for new
  detector types in v1.x.
- **Atomic write is the default for crash-critical writes.** New
  command code that writes manifest-shaped data must go through
  `atomicWriteFileSync` / `atomicWriteJsonSync`.
- **Telemetry events never grow forbidden fields.** Adding any
  field to `TelemetryEvent` requires bumping `v:` and updating
  `docs/telemetry-transparency.md` AND
  `tests/telemetry.test.ts`'s forbidden-fields assertions.
- **Standards docs follow Scope / Rules / Examples / Rationale.**
  New docs added in v1.x should match this shape.

### Gotchas

- **Script-detector worker harness is at `dist/audit/detectors/
  script-worker.js`** post-build. When running tests via vitest
  from `src/`, the script.ts code falls back to walking up to
  package root to find the dist/ artefact. Don't break this lookup;
  it's how the integration suite finds the harness.
- **Ripgrep detector default excludes are baked in.** If a rule
  wants to scan `node_modules/` explicitly, it must pass
  `exclude: []` — not omitting the field. Documented in the
  detector contract.
- **The `--telemetry-endpoint` POST is fire-and-forget.** A slow
  collector cannot slow cadence exit. Testing this end-to-end
  requires either spinning up a real HTTP server in tests or
  mocking `fetch`; neither is wired at v1.0 (the local-log path
  is the load-bearing one).
- **Three-way merge file format depends on templates-history
  presence.** When the v1.0.0 snapshot exists for the entry's
  `templateVersion`, the merge file uses git-style markers
  (`<<<<<<<` etc.). Otherwise it falls back to the v0.5 shape
  (just the new template). The upgrade-test assertion was updated
  to accept either.
- **CHANGELOG.md is the v1.0 release source of truth.** Future
  releases append above the `## [1.0.0]` entry.
