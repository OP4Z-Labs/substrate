## Bugs found during fix work

### Bug 1 — Smoke script: `tar | grep -q` under `set -o pipefail` (FIXED)

**Surface:** Scenario 10f's check `tar -tzf "${TARBALL}" | grep -qE
"^package/CHANGELOG\.md$"` returned nonzero (exit 141) under
`set -o pipefail` even when the pattern existed. With pipefail
enabled, `grep -q` exits early after the first match, the kernel
sends SIGPIPE (13) to tar, and the whole pipeline's exit status
becomes 141 (128+13).

**Fix:** Materialize the tar listing into a local variable first,
then `echo "$listing" | grep -q ...`. Lines 906-914 of
enterprise-smoke.sh.

**Severity:** Smoke-blocker (false negative).

**Lessons:** Any future scenario checking specific tarball contents
needs the same pattern. The smoke handoff for alpha.1 didn't catch
this because the scenario that needed the check (CHANGELOG presence)
wasn't added until beta.1.

### Bug 2 — Smoke handoff inaccuracy: clear-cache --json (NOT A BUG)

**Surface:** Alpha.1 smoke HANDOFF reported bug #6: "`extends
clear-cache --json` doesn't accept a `--json` flag — it just prints
... to stdout." 

**Reality:** The alpha.1 code DID accept `--json` (verified by reading
src/v2/deterministic/extends-command.ts:342-361 and running the binary
manually). The HANDOFF was incorrect on this point.

**Action taken:** No code change. Smoke scenario 10f locks the JSON
envelope as a regression gate.

## Notes for the next agent

1. **The smoke script is now the load-bearing regression gate.** Any
   future change touching extends-awareness MUST run
   `npm run smoke:enterprise` and see 16/16 — that's stronger than
   `npm test` alone catches (the tests use the merge wrappers
   directly; the smoke proves the CLI surface is wired through).

2. **`scripts/copy-changelog.mjs` is the prepack hook.** It runs
   automatically on `npm pack` and copies the workspace-root
   CHANGELOG into packages/substrate/. The staged copy is gitignored.
   If you bump the version, the prepack picks up the new CHANGELOG
   content automatically.

3. **`set -o pipefail` + `grep -q`** in bash is a footgun. If you add
   any scenario checking tarball contents, use the
   `tar_listing="$(tar -tzf "${TARBALL}")"` then `echo | grep`
   pattern. See scenario 10f for the working shape.

4. **`extends-opt-out` is at the top level, not under `extends`.**
   The brief said "`extends.opt-out` field" but the shape that
   shipped is `"extends-opt-out": []` at the config root. Nesting it
   under `extends` would have required restructuring the existing
   `extends: []` array shape, which would have been a breaking
   change for alpha.1 consumers. The top-level form is purely
   additive.

5. **The `--include-opt-out` flag** only lives on `substrate extends
   list` for now. If a future surface needs the same toggle (e.g.
   `substrate audit --include-opt-out`), thread the
   `includeOptOut: boolean` field down through `resolveExtendsChain`
   (already supports it).

6. **No transitive extends.** Beta.1 still only walks one level. If
   an extends source has its own `extends`, the inner one is
   silently ignored (scenario 10d verifies the non-crash). Adding
   transitive support is a v3.0-rc.1 item — would need cycle
   detection + a depth cap.

7. **The github cache slug stays human-readable.** Plan §2.5
   suggested a SHA hash; alpha.1 + beta.1 keep `<org>-<repo>@<ref>`
   for grep-ability. The cache manifest records the resolved SHA so
   reproducibility is preserved. Documented in github-source.ts:108.

8. **No pushes.** This branch is local-only. Beau is sorting the
   `op4z-labs` GH org separately; publishing logistics happen at
   v3.0.0 GA, not beta.1.
