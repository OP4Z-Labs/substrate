#### Finding 6 ‚Äî `pattern`/`paths` shorthand flagged "unknown field"

- **What:** RULES.yaml entries with rule-level `pattern:` (+ optional `paths:`, `exclude:`, etc.) were raising `warn: rules[N] (ID) has unknown field "pattern"`. The canonical form `detector: { type: ripgrep, pattern, paths }` was the only one recognized.
- **Design decision (documented in commit body):** widen the loader rather than narrow the templates because (a) the canonical form is already documented + present in every shipped template ‚Äî no breaking change for existing files, (b) the "unknown field" warning was load-bearing for typo detection, so keeping the allow-list explicit preserves that signal, (c) the shorthand is a real shape in OP4Z's own RULES history.
- **Behavior:** a rule with `pattern:` (and no explicit `detector:` block) is now treated as syntactic sugar for `detector: { type: ripgrep, pattern, paths, exclude, ‚Ä¶ }`. When both are present, `detector:` wins (the explicit form is intentional and may differ).
- **Files:** `src/audit/rules.ts` (added the shorthand branch + widened the `allowed` field set); `tests/audit-runtime.test.ts` (2 regression tests ‚Äî shorthand path + detector-wins precedence).
- **Commit:** `eeeb428 fix(rules): accept rule-level pattern/paths as ripgrep shorthand`.

#### Finding 7 ‚Äî `audit --json` error path emitted human text

- **What:** `substrate audit --json` on a fresh repo (no RULES.yaml) printed `‚úó Substrate: no RULES.yaml found‚Ä¶` to stderr and emitted nothing on stdout. CI callers parsing `--json` got an empty stdout.
- **Fix:** introduced a `JsonAlreadyEmittedError` sentinel exported from `audit.ts`. The audit path now:
  1. Writes the envelope `{ "ok": false, "error": { "code": "rules-not-found", "message": "‚Ä¶" } }` to stdout.
  2. Sets `process.exitCode = 1`.
  3. Throws the sentinel.
- `main()` in `cli.ts` detects the sentinel and skips the usual `console.error("‚úó ‚Ä¶")` line ‚Äî so naive `JSON.parse(stdout)` consumers don't trip over a trailing stderr line.
- **Files:** `src/commands/audit.ts` (sentinel class + JSON branch); `src/cli.ts` (sentinel detection in `main()`); `tests/integration/cli-exit-codes.test.ts` (regression test parses the JSON envelope and asserts shape).
- **Commit:** `71cbcb6 fix(cli): JSON error envelope on audit --json + non-zero exit on hooks describe`.

## Additional observations (not fixed per hard rule #7)

These were noticed while patching but were out of scope for the smoke pass:

1. **Several bundled RULES.yaml detectors emit `ripgrep` regex errors** because the canonical catalog uses look-around regexes (`(?:‚Ä¶)`, `(?!‚Ä¶)`) that ripgrep can't parse without `--pcre2`. Examples: `BE-APIV-001` (`/(?!api/v)`), `FE-TS-001` (`(?!\s*//‚Ä¶)`), `XCUT-MD-001` ("the literal '\\n' is not allowed in a regex"). These fire on every `substrate audit` against a fresh init. Not a publish blocker, but a follow-up to either (a) re-author the patterns without look-around or (b) add a `--pcre2` flag to the ripgrep detector.
2. **`workflow describe` for v2 workflows compresses information.** The legacy `[prompt|command|audit]` pill set can't faithfully represent v2 step types like `gate`, `invoke-sub-workflow`, `prompt-and-action`. The fix mapper does the best it can but a richer v2-aware describe surface (`substrate workflow describe --v2`) would be more honest.
3. **The doctor warning about `auto/.substrate-manifest.json` missing on a fresh init.** When init runs cleanly, the manifest is created ‚Äî but the doctor's warning fires when the file isn't there, suggesting users run `substrate init`. Slightly confusing if you just ran init. Not load-bearing.
4. **`substrate scheduler --check` writes nothing on a freshly-init'd repo** because no workflow declares `trigger: schedule`. This is correct behavior but the empty output `(no scheduled workflows)` could be more helpful (point at the tackle-task workflow's `trigger:` block as an example).

None of these were fixed. Flagged for the next pass.

## Decisions locked

1. **`knowledge-sources.yaml` is intentionally absent after `substrate init`.** The absent file is a first-class state in the loader (`manifestPath: null` triggers the v1 docker-compose + .env.example fallback). Writing a commented-out stub would force every consumer to either delete it or live with a YAML file that contains no meaningful sources.
2. **`pattern` / `paths` are rule-level shorthand for a ripgrep detector**, not deprecated. Documented in `src/audit/rules.ts` inline + in the commit body.
3. **v2 step types map to v1 `command|audit|prompt` for the legacy `workflow describe` surface.** The mapping is preserved verbatim in `src/commands/workflow.ts#normalizeStep`. A future `--v2` flag could expose the richer types.
4. **`JsonAlreadyEmittedError` is the sentinel for `--json` error paths** that need to suppress the human-readable error line in `main()`. The pattern is reusable for any future command that wants the same envelope semantics.
